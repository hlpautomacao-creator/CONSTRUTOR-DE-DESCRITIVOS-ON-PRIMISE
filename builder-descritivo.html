SEÇÃO 04
Fluxos (Rotas)
Monte cada fluxo selecionando os PCs na sequência. O código do fluxo é enviado pelo ERP no pré-cadastro do veículo.
Os fluxos determinam os acessos permitidos e a sequência de operações de cada tipo de veículo. O ticket é encerrado automaticamente no último ponto do fluxo.

SEÇÃO 05
Integração com ERP
Tipo de integração e detalhes técnicos da interface entre Guardian e o ERP do cliente.

Tipo de Integração

🔗 Webservice (SOAP/REST)
ERP chama métodos do Guardian via WebService. Modo passivo e/ou ativo REST/JSON.
📄 Arquivo Texto (SIG)
Integração via arquivos TXT em pasta de rede. Serviço SIG do Guardian.
🗄 Banco Tanque
Integração via tabelas de banco de dados intermediário acordadas com o cliente.

Métodos Webservice utilizados

✓
CadastraTicketGuardian
Importa pré-cadastro do veículo enviado pelo ERP na chegada.
✓
AlteraTicketGuardian
Atualiza dados do ticket após cadastro inicial (exceto cancelados).
✓
ManutencaoTicket — Cancelamento
Altera o estado do ticket. Usado para cancelar antes do veículo acessar.
✓
ManutencaoTicket — Bloqueio/Desbloqueio
Bloqueia o ticket aguardando emissão de NF; ERP desbloqueia após emissão.
✓
ConsultaTicketGuardian
Consulta estado atual de um ticket no Guardian pelo ERP.
✓
Exportação ativa (REST/JSON)
Guardian envia dados de pesagem encerrada ao ERP via Webservice ativo.
